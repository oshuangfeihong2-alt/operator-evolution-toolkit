# 🎲 概率决策增强模块 (Bayesian Decision Enhancement)

> **7本贝叶斯/概率决策经典 → 注入四个现有Skill · 从"凭直觉搏"到"算概率赢"**
>
> 一句话：**每做一个商业决策，你不再问"这样对不对"，而问"在已知信息下，我的胜率是多少？什么新信息会让我改变判断？"**

## 🎯 v2.0 创业阶段适配层

> 解决大企业方案误用到小企业的根本问题。一切建议发放之前，先问阶段。

### 四阶段速诊

| 阶段 | 标志 | 核心任务 | 不该用 | 该用 |
|---|---|---|---|---|
| 0·验证期 | 1-3人，无收入 | 找PMF | OKR/绩效/事业部 | 精益画布/Mom Test/MVP |
| 1·生存期 | 3-10人，有收入不稳 | 跑通现金流 | 规模化架构 | 跨越鸿沟/增长黑客/格鲁夫杠杆率 |
| 2·增长期 | 10-50人，稳定增长 | 建团队建流程 | 单打独斗技巧 | 德鲁克用人所长/1-on-1/绩效 |
| 3·规模化 | 50人+，模式成熟 | 体系+第二曲线 | — | 全框架 |

### 德鲁克争议裁决

《卓有成效的管理者》的时间管理/要事优先/有效决策=全阶段适用。
事业部制/战略规划/管理层级设计=阶段2+。阶段0-1看了没用，Skill不会推。

### 诊断触发词

你说刚起步/一个人/还没产品/有客户了/在招人/扩团队/团队大了，Skill自动先判断阶段再开药。

## 这个模块的本质

贝叶斯思维不是一项独立技能。它是一种**底层决策操作系统**。把它注入四个Skill后：

| 注入前 | 注入后 |
|---|---|
| 做决策看结果好坏 | 做决策看**决策过程质量**（结果≠质量） |
| "这个项目该不该做？"（二元） | "在做这个项目的条件下，成功的概率是X%，我赌吗？"（概率） |
| 失败了就放弃了 | 失败了问"这说明我的什么假设错了？"（信念更新） |
| 坚持是对的，放弃是失败 | **主动放弃是策略**（止损=释放机会成本） |
| 信自己看到的 | 信**外视角+基础概率**（自己看到的不可靠） |

## 四个Skill的注入位置与内容

### 注入①：经营者操作系统 · 新增「L2.5 概率决策层」

在 L2(行动引擎) 与 L3(系统思考) 之间插入：

#### 2.5.1 决策质量 ≠ 结果质量（《对赌》核心）

```
                 好结果          坏结果
              ┌──────────┬──────────┐
    好决策    │ 应得的奖励│ 坏运气    │  ← 别在这里放弃好决策
              ├──────────┼──────────┤
    坏决策    │ 好运气    │ 咎由自取  │  ← 别在这里重复坏决策
              └──────────┴──────────┘

单一结果说明不了任何问题。
好决策 = 在不确定条件下，选择了最有利于目标的选项（考虑可能性+回报+概率）。
```

#### 2.5.2 德鲁克有效决策五要素

1. **界定问题类型**：这是"通用问题"（需要原则/标准流程解决）还是"特例"（需个案处理）？ 大多数你以为是特例的事其实只是旧问题换了新包装
2. **明确边界条件**：决策必须满足的最低目标是什么？什么条件必须为真？
3. **先想"什么是正确的"**，再想"谁妥协多少"——不从可接受性往回推
4. **把执行行动融入决策**：谁、做什么、何时、谁来检查？不化为具体行动的决策=愿望
5. **建立反馈机制**：做决策时就约定好什么时候用什么指标检验。如果假设错了，什么时候止损？

#### 2.5.3 反脆弱系统设计（《反脆弱》核心）

```
三元结构：
  脆弱 ──── 强韧 ──── 反脆弱
  (波动坏)    (波动无关)    (波动好)

杠铃策略：
  90%放在极度安全的地方
  10%放在高风险高回报的地方
  中间地带（中等风险中等回报）全部砍掉
  → 损失有上限，收益无上限

可选性(Optionality)：
  做更多"对了收益巨大，错了损失有限"的事
  不是预测未来，是让自己在未来不管发生什么都能活得好
```

#### 2.5.4 10-10-10 幸福感测验

```
做决策前问：
  10分钟后我会怎么看待这个选择？
  10个月后呢？
  10年后呢？
→ 如果10年后几乎没有影响，这就是一个低风险决策——快快决定。
```

### 注入②：商业模式策划与人性设计 · 升级 L5(人性密码) 和新增 L7(概率验证层)

#### L5 增强：芒格人类误判心理学25条（补充现有行为经济学）

从25条中选出与商业模式最相关的12条（现有行为经济学层已有锚定/损失厌恶/框架效应，以下为补充）：

| # | 心理倾向 | 对商业模式的启示 |
|---|---|---|
| 1 | 激励超级反应 | "给我看激励，我就告诉你结果。"错误的激励设计=模式自毁 |
| 2 | 喜爱/热爱倾向 | 用户一旦认同你的品牌，会忽略缺点、扭曲事实——积累品牌资产的值钱 |
| 3 | 讨厌/憎恨倾向 | 一次糟糕体验让用户永久低估你的所有优点 |
| 4 | 避免怀疑倾向 | 用户困惑=什么都不做。"是否"选择远好于"选哪个" |
| 5 | 避免不一致倾向 | 一旦用户做了小承诺，维持一致的动力极强（=承诺一致原则的强化版） |
| 6 | 艳羡/妒忌倾向 | "别人有了我也要有"——稀缺+社会认同的双引擎 |
| 7 | 回馈倾向 | 不只是互惠，还包括以德报德、以牙还牙的强烈冲动 |
| 8 | 简单联想谬误 | 用户会把"包装好看"联想成"产品质量好" |
| 9 | 自视过高倾向 | 用户会高估自己的能力——"我当然能坚持每天用这个App" |
| 10 | 过度乐观倾向 | 用户（和创始人）都会系统性低估风险——做最坏打算 |
| 11 | 被剥夺超级反应 | 失去已有的比从未得到更痛苦（免费试用后取消=痛苦） |
| 12 | 社会证据倾向 | 看到别人做了，就会觉得"这样做是对的" |

#### 新增 L7 · 概率验证层

**7.1 贝叶斯信念更新循环**

```
你的商业模式基于一个核心假设。
这个假设不是"对"或"错"——它有一个概率。

  先验概率（你做之前认为的） 
       ↓ 拿到新证据
  后验概率（更新后的判断）
       ↓ 持续迭代
  越来越接近真相

关键操作：每一步都问——"什么证据会让我更新这个概率？"
```

**7.2 止损决策矩阵（《停损的胜算》）**

```
                   坚持正确          放弃正确
                 (应得的奖励)      (错误放弃)
              ┌──────────────┬──────────────┐
    你坚持了   │              │              │
              ├──────────────┼──────────────┤
    你放弃了   │              │              │
              │ (后悔)        │ (正确止损)    │
              └──────────────┴──────────────┘

止损判断三步：
1. 设定终止条件（在开始之前）——"什么客观条件触发时我放弃？"
2. 区分"猴子"（具体问题）和"基座"（根本问题）——修猴子不需要放弃基座
3. 用基础概率做参照——类似项目在全行业的成功率是多少？
```

**7.3 事前验尸（Pre-mortem）**

```
假设现在是2年后，这个项目失败了。
"我们是因为什么失败的？"
→ 团队每人匿名写3条最可能的失败原因
→ 汇总→排序→针对Top 3提前设防火墙

这个练习做了以后，失败概率不一定降到零，
但你对失败的准备程度和反应速度会上一个量级。
```

### 注入③：权力博弈手册 · 升级 L2(谈判武器)

#### L2 增强：扑克式博弈框架

**谈判=扑克，不是象棋**

```
象棋思维 ❌              扑克思维 ✅
─────────────────    ─────────────────
对方"应该"理性        对方可能虚张声势
每一步有最优解         没有最优解，只有范围
追求确定性             管理不确定性
一局定胜负             长期+EV才是关键
信息完全透明           信息不对称是常态
```

**谈判中的概率四步**：

1. **范围分析**："对方这一步是什么意思？列出所有可能的解释范围（不是猜单一答案）"
2. **概率分配**："在我的范围内，他真正想要X的概率是___%，想要Y的概率是___%"
3. **期望值计算**："如果我出策略A，面对他的每种真实意图，我的期望收益是多少？"
4. **信息获取**："什么信息能让我缩小对他的猜测范围？我该怎么获取？"

**BATNA的概率化升级**：

```
传统BATNA = "谈崩了怎么办"
概率化BATNA = "谈崩了以后，我的各种选项各自成功概率是多少，期望值是多少"

不是"我的BATNA是找B公司合作"
而是"找B公司：成功概率60%，利润100万；找C公司：成功概率80%，利润60万
      → BATNA期望值 = 108万 → 低于这个数的offer我拒绝"
```

### 注入④：沟通说服引擎 · 升级 L5(应用层)

#### L5 增强：芒格说服心理学补充

关键对话中应用芒格智慧：
- **反过来想**：不是"怎么让他同意"，而是"为什么他可能不同意？把所有反对理由列出来解决掉"
- **能力圈诚实**："这个我不确定" > 瞎编——对方对你"知道自己不知道什么"的信任 > 对你"假装什么都懂"的信任
- **避免不一致倾向在话术中的应用**：先让对方说出一个与你目标一致的小承诺，再逐步引导
