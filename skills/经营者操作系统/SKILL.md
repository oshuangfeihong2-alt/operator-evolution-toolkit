## L1 · 方向导航层

### 1.1 核心三问（任何事先问这三个问题）

```
Q1: 这件事的"终"是什么？我期望的最终结果是什么？
    ← 源自《高效能人士的七个习惯》习惯二：以终为始
    ← 源自《搞定》："你期望的结果是什么？"

Q2: 我能直接控制/间接影响/还是只能接受？
    ← 源自《高效能人士的七个习惯》影响圈/关注圈

Q3: 这件事在我的刺猬理念三环交集内吗？
    ← 源自《从优秀到卓越》刺猬理念
      一环：我擅长什么？（能力）
      二环：我热爱什么？（热情）
      三环：能创造什么价值？（经济引擎）
```

### 1.2 刺猬理念定位法

**使用时机**：面临重大选择、职业方向迷茫、决定要不要接一个任务时。

**操作步骤**：
1. 画三个相交的圆
2. 圆1：写下你比大多数人做得好的事（不是"喜欢"，是"擅长"）
3. 圆2：写下你做起来不觉得累、甚至享受的事
4. 圆3：写下你的工作/角色最需要创造的价值是什么
5. 三环交集区 = 你的战略定位
6. **凡是三环外的事务，学会说"不"**

### 1.3 积极主动转换器

**遇到负面情绪时，做这个语言替换**：

| 受害者语言（❌） | 主动者语言（✅） |
|---|---|
| "他把我气疯了" | "我选择如何回应他的行为" |
| "我不得不加班" | "我选择优先完成这件事" |
| "要是老板不这样就好了" | "我可以做什么来改善这个局面" |
| "我就是这样的人" | "我可以学习不同的做法" |
| "没办法，只能这样" | "让我试试看别的办法" |



### 1.5 精要主义：做更少但更好（v1.3 补充）

> 来源：《精要主义》格雷戈·麦吉沃恩

**精要主义三步法**：
1. 探索：花时间分辨什么才是真正重要的（不是"我能做什么"，而是"我应该做什么"）
2. 排除：对不重要的事说"不"——"如果还不是明确的'是'，那就是明确的'不'"
3. 执行：用最小阻力让重要的事变得毫不费力

**精要主义者的核心思维**：
- "更少但更好"——不是完成更多事，是完成对的事
- 学会说"不"而不得罪人——"我现在手上有三个项目，再接一个会影响所有项目的质量"
- 取舍是真实的——做了A就做不了B，接受这个事实而非假装两全

**每天问自己**："我现在在做的事，是唯一重要的那件事吗？"

## L2 · 行动引擎层

### 2.1 GTD五步工作流（横向管理核心）

```
收集 ──→ 理清 ──→ 组织 ──→ 回顾 ──→ 执行
 │         │         │         │         │
 │   100%   │  两个   │  按情境  │  每周   │  四标准
 │   捕获   │  核心问 │  分类存储 │  回顾   │  选行动
```

**第一步：收集**
- 拿出纸笔或打开笔记软件
- 把脑子里所有"该做/想做/可能需要做"的事全部写下来
- 规则：不判断、不分析、不排序、不遗漏
- 范围：工作+个人，大小事务全部算上
- 工具提示清单帮你覆盖死角：邮件、项目、会议、财务、健康、家庭、学习...

**第二步：理清（核心两问）**
- 面对每一项："这是什么？"
- "需要采取行动吗？"
  - 否 → 垃圾/将来也许/参考资料
  - 是 → **"下一步行动是什么？"**（必须是具体可执行的动词开头）
  - 是 → **"我承诺完成什么结果？"**（如果是多步骤＝项目）

**第三步：组织**
- 日程表：只放有刚性时间要求的事（硬质景观原则）
- 下一步行动清单：按情境分类（@电话 @电脑 @办公室 @家 @外出）
- 项目清单：所有需要多步骤完成的结果（一行一个，30-100个正常）
- 等待清单：委派出去的，标注"谁+什么事+何时"
- 将来/也许清单：现在不做但以后可能做的

**第四步：回顾（GTD的灵魂）**
- **每周回顾是底线**，不可跳过
- 三件事：确认（清空收集篮）→ 更新（过一遍所有清单）→ 创造（看将来/也许，激发新想法）
- 时间：每周五下午或周末固定时段，1-2小时

**第五步：执行（四标准选行动）**
1. 情境：你在哪？手头有什么工具？
2. 时间：你有多少可支配时间？
3. 精力：你现在的能量水平？
4. 重要性：在筛选后的选项中，哪个回报最高？

### 2.2 时间管理矩阵（要事第一）

```
        紧急           不紧急
      ┌──────────┬──────────┐
重要  │ Q1 危机  │ Q2 规划  │  ← 高效能人士的"甜蜜区"
      │   迫切问题│   学习成长│
      ├──────────┼──────────┤
不重要│ Q3 干扰  │ Q4 浪费  │
      │   不速之客│   无意义 │
      └──────────┴──────────┘
```

**核心原则**：
- Q2（重要不紧急）是高效能生活的核心——把时间投在这里，Q1自然缩小
- 如果90%时间在Q1，你永远被危机驱动——这说明Q2欠债了
- Q3的事情学会委派或拒绝
- Q4的事情立刻停止

### 2.3 周计划法（第四代时间管理实战）

**每周30分钟，按这四步走**：

```
步骤1：确认下周关键角色（限制在7个以内）
       例：[技术负责人] [下属的导师] [丈夫] [个人学习者]

步骤2：为每个角色定1-2个Q2目标
       例：技术负责人 → 完成系统架构评审文档
            个人学习者 → 读完《格局》前三章

步骤3：为每个目标在周历上安排时间块
       ★ 以大块时间（2-3小时）为单位，不是碎片15分钟

步骤4：每天早上花5分钟根据当天情况微调
       灵活应变，但周目标不变
```

### 2.4 两分钟原则

**处理任何新任务时**：如果能两分钟内完成，**立刻做**。
- 回复简短邮件、确认时间、转发文件、丢弃垃圾邮件...
- 这个原则每天帮你节省1小时以上的"重新找到它再处理"的时间



### 2.6 深度工作：专注力是新的超能力（v1.3 补充）

> 来源：《深度工作》卡尔·纽波特

**深度工作 vs 浮浅工作**：
- 深度工作：在无干扰状态下专注进行的高认知活动，产出难以复制
- 浮浅工作：在分心状态下进行的低认知、容易复制的事务（回邮件、开会、刷消息）

**深度工作四准则**：
1. 深度工作：选择你的深度哲学（禁欲式/双峰式/节奏式/记者式——4种任选）
2. 拥抱无聊：训练大脑远离分心。排队不看手机、散步不带耳机
3. 退出社交媒体：不是戒网，是"只在证明价值时才保留一个工具"
4. 砍掉浮浅：固定每天浮浅工作的上限（比如不超过2小时）

**注意力残留效应**：当你从任务A切换到任务B，A的残留注意力还在大脑里运行。频繁切换=大脑永远不清醒。

### 2.5 自然计划模式（纵向项目规划）

**面对需要深度思考的项目，用这五步**：

```
1. 定义目的和原则 → 为什么做这件事？成功的标准是什么？
2. 展望结果       → 大功告成时是什么样的？听上去、感觉如何？
3. 头脑风暴       → 关于这件事的所有想法全写下来，不判断不筛选
4. 组织整理       → 排序、分组、找出先后依赖关系
5. 明确下一步行动  → 每个构件定一个具体的、可执行的下一步
```

### 2.5.5 创业维艰：该坚持还是该放弃（v1.3 补充）

> 来源：《创业维艰》本·霍洛维茨

**战时CEO vs 和平CEO**：
- 和平CEO：按流程来，该授权授权，该建文化建文化
- 战时CEO：公司快死了，什么流程都不重要，活下来是唯一的目标
→ 你先判断自己处于哪种状态，再决定用哪种打法

**CEO最难的两件事**：
1. 把正确的人放在正确的位置上（比做产品难）
2. 在自己害怕得要死的时候，还能让团队觉得一切都在掌控之中

**该坚持还是放弃？霍洛维茨判断法**：
- 不是问"这个项目能不能成"——而是问"如果它能成，我还想在这里做吗？"
- 如果答案是"是"——坚持。如果答案是"不想"——走。

## L3 · 系统思考层

### 3.1 看问题从"点"到"面"：画系统循环图

**什么时候用**：一个问题反复出现、解决方案带来新问题、感觉"越努力越糟糕"时。

**画图12条黄金法则（精简核心）**：

```
1. 定义边界 → 什么在系统内？什么在系统外？（用悬摆标出）
2. 从有趣的地方开始 → 最关键的外部驱动力或核心成果
3. 问两个问题 → "它驱动什么？" "什么驱动它？"
4. 用名词不用动词 → "服务质量"而非"提高服务"
5. S型连接 = 同向（原因↑结果↑）/ O型连接 = 反向（原因↑结果↓）
6. 数O型连接：偶数个O = 增强回路（指数增长或崩溃）
               奇数个O = 调节回路（维持目标，可能振荡）
7. 标注延迟：用"//"标记存在时间延迟的连接
8. 不要爱上你的图 → 扔掉重画是正常的，关键是反映实况
```

### 3.2 六大高杠杆系统基模（速查卡）

这是最常见的六种系统困境，每种都附带**杠杆解**：

| 基模 | 症状 | 错误的做法 | ✅ 杠杆解 |
|---|---|---|---|
| **增长极限** | 努力很久但效果递减 | 更努力推增长引擎 | **消除限制因素**，不是更用力推 |
| **转移负担** | 每次都走捷径，问题越来越严重 | 继续用症状缓解法 | **聚焦根本解**。捷径只用来争取时间 |
| **目标侵蚀** | 标准不知不觉在降低 | 接受下降后的"新常态" | **死守愿景**。坚持原来的目标 |
| **恶性竞争** | 双方不断升级对抗 | 继续升级投入 | **一方主动逆转**或找到双赢路径 |
| **饮鸩止渴** | 短期修补带来长期更严重问题 | 继续用同类修补 | **关注长期后果**，戒断症状依赖 |
| **公地悲剧** | 每个人多拿一点，最后资源耗尽 | 继续各自为政 | **建立公共规则**和自律机制 |

### 3.3 底层逻辑快速校验（5个关键检查点）

做任何重要决策前，过这5个检查点：

```
1. 是非对错 → 这件事的根本逻辑是什么？有没有隐藏的假设？
2. 思考维度 → 我从什么角度看？换一个角度会看到什么？
3. 个体进化 → 这件事能帮我在哪个维度成长？
4. 理解他人 → 对方的利益和动机是什么？（不是他"应该"怎样）
5. 社会协作 → 谁从中受益？谁承担成本？系统效率如何？
```

### 3.4 深度思维工具箱

| 场景 | 工具 | 用法 |
|---|---|---|
| 分析问题原因 | **5Why法** | 连问5个为什么，挖到根本原因 |
| 预测结果 | **逻辑链延伸** | "如果A发生→B可能→C可能→..."延伸至少3步 |
| 理解对立面 | **换位思维** | 把自己放在对方的角色、信息、利益里想一遍 |
| 复杂问题可视化 | **思维导图/流程图** | 把模糊变清晰，把隐含假设全部摊开来 |
| 找到关键因素 | **二八法则** | 20%的什么因素导致了80%的结果？ |

### 3.5 五项修炼诊断清单

当你觉得组织问题频出时，对照检查：

```
□ 系统思考：大家是否看到事件之间的连接？还是只看到孤立事件？
□ 自我超越：团队成员有明确的个人愿景吗？还是只是"打工"？
□ 心智模式：有没有"不可讨论"的话题？有没有"声称的理论≠实行的理论"？
□ 共同愿景：大家往同一个方向走吗？还是各管各的？
□ 团队学习：团队思考质量是否超过个人之和？还是会开完了等于没开？
```

## L4 · 关系杠杆层

### 4.1 向上管理：从"等安排"到"管理老板"

**主动汇报三层次**：

```
层次1 · 日常同步：
  "杨总，本周我的三个重点是：①___ ②___ ③___。目前进度：___。需要您决策的：___。"
  → 每周一早上发，不用等老板问

层次2 · 关键节点汇报：
  "杨总，项目X到了里程碑Y，目前结果是___，与预期比___，我的建议是___，需要您___。"
  → 每个关键节点主动汇报，不等到deadline

层次3 · 方案式汇报（最高级）：
  "杨总，关于问题X，我分析了三种方案：
   A方案：___（优点/缺点/成本/风险）
   B方案：___（同上）
   C方案：___（同上）
   我推荐B方案，因为___。您觉得呢？"
  → 不是把问题抛给老板，而是带方案找老板
```

**向上汇报禁忌（反模式）**：
- ❌ 只说问题不给方案（"老板，这个有问题"——然后呢？）
- ❌ 只报喜不报忧（最后爆雷了老板才知道）
- ❌ 等老板问才说进度（老板以为你在摸鱼）
- ❌ 越级汇报但不抄送直属领导

### 4.2 横向领导力：没有权力也能推动协作

**即便没有上下级关系，你也可以用这五步推动协作**：

```
1. 明确共同目标 → "我们都想让X成功，对吗？"
2. 系统化思考 → 画出谁做什么、何时、依赖关系
3. 持续学习 → "上次我们哪里做得好？哪里可以改进？"
4. 全力投入 → 以身作则，先把自己的部分做到位
5. 提供反馈 → 及时、具体、对事不对人
```

**平级协作核心原则**：
- 先存款再提款（情感账户：先帮对方，再开口请帮忙）
- 提问而非命令（"你觉得这样会不会更好？"而非"你应该这样做"）
- 让对方有选择权（"有三个方案，你更倾向哪个？"）

### 4.3 情感账户操作手册

**七种存款方式**：

| 存款行为 | 具体做法 |
|---|---|
| ① 理解他人 | 了解对方真正在乎什么，用他接受的方式对待他 |
| ② 注意小节 | 守时、记住名字、说谢谢、回复消息不拖延 |
| ③ 信守承诺 | 答应了就一定做到。做不到提前解释并重新协商 |
| ④ 明确期望 | 一开始就说清楚"我想要什么、什么算完成" |
| ⑤ 正直诚信 | 背后维护不在场的人。对所有人心口如一 |
| ⑥ 勇于致歉 | 错了就认，不狡辩。但注意：一再道歉就贬值了 |
| ⑦ 无条件支持 | 不以对方的表现为条件——"你做没做好我都站在你这边" |

**五种提款行为（必须避免）**：
粗鲁无礼 · 失信违约 · 背后攻击 · 无视对方 · 期望不明确

### 4.4 会议必问终结问题

**任何会议/讨论结束前，必须明确**：
> "那么，下一步行动是什么？谁来做？什么时候完成？"

就这一句话，能把90%的"会开完了什么都没发生"变成"会后有行动"。

## L5 · 商业思维层

### 5.1 商业本质六问（杰克·韦尔奇框架）

做任何商业决策前，先回答这六个问题：

```
1. 我们面对什么市场？客户真正需要什么？
2. 我们的竞争优势是什么？（凭什么客户选我们？）
3. 财务基本面如何？（收入、成本、利润、现金流）
4. 团队能力匹配吗？（人对了没有？）
5. 战略是什么？（做什么、不做什么——"不做什么"更重要）
6. 执行节奏？（多快？关键里程碑是什么？）
```

### 5.2 精益创业MVP验证法

**当你要推动一个新项目/新想法时**：

```
Step 1: 定义核心假设
        "要想这个项目成功，哪个假设必须为真？"
        （例如："客户愿意为更快的服务多付20%"）

Step 2: 设计MVP（最小可行产品）
        用最小的成本和最快的速度做出能检验那个假设的东西
        （不是做完整产品，是做"刚好够验证假设"的东西）

Step 3: 构建-测量-学习循环
        做 → 放出去 → 看数据 → 学到什么 → 调整或坚持

Step 4: 转型还是坚持？
        - 假设成立 → 坚持，扩大
        - 假设不成立 → 转型（不是失败，是学到了）
```

**核心原则**：
- MVP的"最小"是真的最小——可能只是一个网页、一个服务原型、一个手工流程
- 虚荣指标不算数（"注册用户数"不重要，"活跃使用率"才重要）
- 失败越快越便宜，成功越早越值钱

### 5.3 用数据讲故事（技术人必备）

**把技术成果翻译成商业语言的四步法**：

```
1. 先给结论（不是先给数据）
   "我们应该追加技术投入，预期12个月内ROI超过200%。"
   
2. 再用可视化支持（不是甩出原始数据）
   - 趋势用折线图
   - 对比用柱状图
   - 占比用饼图/环形图
   - 关系用散点图
   
3. 然后给出推理过程（精简，不超过3个支撑点）
   "三个原因：①___ ②___ ③___"
   
4. 最后给出明确建议 + 下一步行动 + 你的承诺
   "我建议下周启动，我可以先做___。"
```

**数据叙事禁忌**：
- ❌ 把一堆数字丢出来让听众自己找结论
- ❌ 饼图超过5块（合并"其他"）
- ❌ 用3D效果图表（扭曲数据感知）
- ❌ 结论模糊（"可能""一定程度上"——给出明确的判断）

## L6 · 心态地基

### 6.1 创始人身份认同（从"打工"到"这是我的事业"）

**打工心态 vs 创始人心态对照**：

| 维度 | 打工心态 ❌ | 创始人心态 ✅ |
|---|---|---|
| 时间观 | 等下班、等周末、等发工资 | 时间是我的投资 |
| 责任观 | "这不是我的事" | "这件事最终是我的事" |
| 问题观 | 发现问题=抱怨 | 发现问题=机会 |
| 学习观 | 公司出钱培训才学 | 为自己的未来学习 |
| 结果观 | 我做了（不管结果） | 我负责直到成果出现 |
| 老板观 | 老板应该给我... | 我能帮老板解决... |

**每日创始人练习**：
> "如果这家公司是我的，我今天会做什么不同的决定？"
> 然后去做那个决定。

### 6.2 内向者职场优势（《安静》核心）

**内向者不需要变成外向者。你的优势是**：

- 深度思考能力 → 在需要系统分析的复杂问题上碾压外向者
- 倾听能力 → 移情聆听是内向者天然擅长的（习惯五的实践者）
- 书面表达 → 用好邮件和文档，让思考深度变成影响力
- 一对一深度关系 → 不需要在饭局上左右逢源，几个关键人物的深度信任就够了

**内向者行动策略**：
1. 重要会议前提前准备（发挥你的优势）
2. 用会后跟进邮件补充发言（内向者最爱的方式）
3. 把"社交"变成"一对一深度对话"（你擅长这个）
4. 每天给自己恢复型独处时间（不是逃避，是保养）

### 6.3 绝对坦率：关心+直接

**四种沟通方式矩阵**：

```
                直接挑战
              弱        强
      ┌──────────┬──────────┐
      │ 毁灭性   │ 绝对坦率 │  ← 理想状态
关心个人│ 同理心   │          │    "我关心你，所以我要直接告诉你"
      ├──────────┼──────────┤
      │ 虚情假意 │ 恶意侵犯 │
      │          │          │
      └──────────┴──────────┘
```

**如何做到绝对坦率**：
1. 先确认关系基础（"我说这些是因为我在乎你的成长"）
2. 批评对事、赞美对人（"这个方案有问题"≠"你有问题"）
3. 私下批评、公开表扬
4. 批评时给具体改进建议，不是只说你哪里错了

## 🧪 快速自检：你现在处于什么状态？

用GTD自我管理矩阵诊断：

```
              ┌─────────────┬─────────────┐
   高视角     │ 空想家      │ 舰长和指挥官 │ ← 目标
              │ (想法多无执行)│ (自主从容)   │
              ├─────────────┼─────────────┤
   低视角     │ 受害者      │ 微观管理者   │
              │ (疲于奔命)   │ (机械执行)   │
              └─────────────┴─────────────┘
               低掌控        高掌控
```

- **受害者**：从L2开始，先搞定行动层面的掌控（大脑清扫→收集→理清→周计划）
- **微观管理者**：从L1开始，建立方向感（使命宣言→刺猬理念→关注大局）
- **空想家**：从L2的GTD收集和下一步行动开始，把想法落地
- **舰长和指挥官**：从L3/L5开始，扩展系统思考和商业思维深度

## ⚠️ 常见全局反模式

| # | 反模式 | 为什么错 | 正确做法 |
|---|---|---|---|
| 1 | **把做事当目的** | 忙≠有效能 | 先问"这件事的终是什么" |
| 2 | **等安排才行动** | 这是执行者心态的核心症状 | 用周计划法主动定义下周最重要的事 |
| 3 | **只看事件不看结构** | 头痛医头，脚痛医脚 | 画出系统循环图，找杠杆解 |
| 4 | **爱做名词不爱做动词** | 说"沟通"但没人知道下一步是什么 | 说"约张三明天下午聊X项目分工" |
| 5 | **跳过第二象限投资** | 永远在灭火 | 每天给Q2留出不可侵犯的时间块 |
| 6 | **自己扛所有事** | 授权变弃权 | 用责任型授权五要素：结果+方针+资源+考核+奖惩 |
| 7 | **完美主义瘫痪** | 等着万事俱备 | MVP先行，边做边学边调整 |
| 8 | **把妥协当双赢** | 1+1=1.5不是双赢 | 找第三条路——比任何一方最初方案都更好的解 |
