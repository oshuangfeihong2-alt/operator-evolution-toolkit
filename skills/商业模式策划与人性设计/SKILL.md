### 1.2 价值主张设计的11个切入点

你的产品/服务在哪个维度创造价值？

```
□ 新颖 —— 满足客户从未有过的需求
□ 性能 —— 更好/更快/更强
□ 定制化 —— 量身定做
□ 把事情做好 —— 帮客户完成特定工作
□ 设计 —— 出众的设计本身就是价值
□ 品牌/身份地位 —— 使用你的产品是身份的象征
□ 价格 —— 更便宜（不是最低价，是最合适的价格）
□ 成本削减 —— 帮客户省钱
□ 风险抑制 —— 帮客户降低风险
□ 可达性 —— 让以前买不起/买不到的人能获得
□ 便利性/可用性 —— 让事情更简单
```

### 1.3 五大商业模式式样速查

| 式样 | 核心逻辑 | 经典案例 |
|---|---|---|
| **非绑定式** | 把产品/客户关系/基础设施拆成三个独立业务 | 私人银行（客户关系）+ 基金公司（产品） |
| **长尾式** | 大量利基产品，每种卖不多但总量可观 | Netflix、亚马逊 |
| **多边平台** | 连接两个以上相互依赖的群体，收"平台税" | 淘宝、滴滴、美团 |
| **免费式** | 一个群体免费，另一个群体付费 | 百度搜索（广告免费）、Freemium |
| **开放式** | 把外部创意引入/把内部闲置资源变现 | P&G Connect+Develop |

## L2 · 战略定位——在哪里赢、凭什么赢

### 2.1 好战略的构成（《好战略坏战略》核心）

```
好战略 = 诊断 + 指导方针 + 连贯行动

诊断：这个生意面临的核心挑战是什么？（一句话说清楚）
指导方针：应对这个挑战的总体方法是什么？（不能用"我们要做得更好"）
连贯行动：为实现指导方针必须做的一系列互相协同的具体动作
```

**坏战略的标志**（发现以下任何一条，说明你的战略需要重写）：
- 满篇空话（"成为行业领先者"——但没有怎么做）
- 把目标当战略（"增长30%"是目标，不是战略）
- 没有聚焦——想什么都做
- 没有诊断——不知道自己面对的真正挑战是什么

### 2.2 竞争定位速查

**波特三种通用战略（选一个，别站中间）**：

| 战略 | 逻辑 | 风险 |
|---|---|---|
| **成本领先** | 我比你便宜 | 价格战→大家一起亏 |
| **差异化** | 我跟你不同 | 差异被模仿→不独特了 |
| **聚焦** | 我只做这一块 | 市场太小→天花板低 |

**蓝海战略四步动作**（同时问四个问题）：
```
剔除：行业里哪些理所当然的要素可以去掉？
减少：哪些要素可以降到行业标准以下？
增加：哪些要素可以提到行业标准以上？
创造：哪些行业从未提供的要素可以创造出来？
→ 四个动作做完，你应该得到一张和所有竞争对手都不同的战略布局图
```

### 2.3 从0到1 vs 从1到N（《从0到1》核心问题）

```
"在什么重要问题上，你跟别人的看法完全不同？"

如果你对这个问题的答案是"没什么不同"——你做的就是从1到N。
如果你有一个反常识的"秘密"——你做的才可能是从0到1。

垄断企业的特征（值得追求的方向）：
✓ 专利技术（比替代品好10倍以上）
✓ 网络效应（用的人越多越有价值）
✓ 规模经济（越大越便宜）
✓ 品牌优势
```

## L3 · 盈利模式——钱到底怎么赚

### 3.1 22种盈利模式精简速查（《发现利润区》核心）

按最常见的商业模式挑出最实用的8种：

| 盈利模式 | 怎么赚 | 适合什么生意 |
|---|---|---|
| **客户解决方案** | 深度了解客户，卖解决方案而非产品 | 咨询、企业服务、医疗 |
| **产品金字塔** | 底部低价走量，顶部高利润 | 手表、汽车、手机 |
| **多组件** | 主机+耗材+服务分别定价 | 打印机+墨盒、剃须刀+刀片 |
| **配电盘** | 做中间人撮合交易抽佣 | 平台、中介、交易所 |
| **速度** | 比别人快一步→溢价 | 时尚、科技、快消 |
| **卖座大片** | 集中资源押注少数几个爆款 | 制药、电影、游戏 |
| **利润乘数** | 一个核心资产多次变现 | 迪士尼IP→电影+玩具+乐园 |
| **创业家** | 极致瘦身降低盈亏平衡点 | 小而美的SaaS、工作室 |

### 3.2 利润区诊断三问

```
Q1：在这个行业里，利润集中在哪一段？
    → 不是每个环节都赚钱。利润可能在设计/品牌/渠道/服务/配件…
    
Q2：你的战略控制指数是多少？（1-10分）
    1分=什么都没有，纯拼价格
    10分=行业标准制定者，离开你不行
    你现在几分？目标几分？

Q3：你在价值链的哪个位置？
    → 离钱最近的地方，利润最厚
    → 离客户最近的地方，话语权最大
```

## L4 · 创新引擎——凭什么不同

### 4.1 颠覆式创新诊断（《创新者的窘境》核心）

```
        ┌──────────────────────┐
        │  延续性创新          │  ← 把好产品做得更好
        │  （在位者擅长）       │     大公司在这个维度碾压你
        ├──────────────────────┤
        │  颠覆式创新          │  ← 在在位者不屑/不愿/不能做的维度切入
        │  （创业者的武器）     │     低端颠覆：服务被过度服务的客户
        │                      │     新市场颠覆：服务未被服务的客户
        └──────────────────────┘

判断你的创新是不是"颠覆式"：
→ 在位者会跟吗？如果他们会跟而且能碾压你，你就不是颠覆
→ 在位者会忽略吗？如果他们会说"这点小市场不值得做"——你是颠覆
```

### 4.2 MVP设计模板（精益创业核心）

```
你要验证的核心假设是什么？
"要想这个模式成功，哪个假设必须为真？"

设计你的MVP：
- 用最小的成本、最快的速度做出"刚好够验证那个假设"的东西
- 不是"做一半功能的产品"，是"做刚好能测试假设的实验"

BML循环：Build（做）→ Measure（测）→ Learn（学）
每一轮循环只验证一个核心假设
循环越快越好——"失败越快越便宜，成功越早越值钱"

关键指标 vs 虚荣指标：
✓ 关键指标：留存率、活跃度、复购率、NPS
✗ 虚荣指标：注册用户数（但不活跃）、下载量（但不打开）
```

### 4.3 紫牛测试（《紫牛》核心）

> "你的产品值得被谈论吗？如果用户不会主动告诉朋友，它就是一头普通的牛。"

```
紫牛四问：
1. 谁看到你的产品会说"哇"？
2. 他们会对谁说？
3. 他们会在什么场合说？
4. 他们说了之后会发生什么？

如果你的答案是"没人会主动说"——你的产品还没有差异化到值得存在。
```

## L5 · 人性密码——用户行为背后的大脑机制

### 5.1 双系统决策模型（《思考，快与慢》核心）

```
系统1（快）          系统2（慢）
─────────────────   ─────────────────
自动的、不费力的      需要集中注意力
直觉的、情绪化的      理性的、分析性的
全天候运行           懒，能不启动就不启动
做大部分日常决策      做重大、复杂决策

对商业设计的启示：
→ 用户90%的购买决策是系统1做的
→ 如果你的产品需要用户启动系统2才能理解——你的转化率会很难看
→ 让"选择你"成为系统1的默认选项
```

### 5.2 行为经济学在商业模式中的应用

| 效应 | 解释 | 怎么用 |
|---|---|---|
| **损失厌恶** | 失去100块的痛苦≈得到200块的快乐 | 免费试用→"不用就亏了" |
| **锚定效应** | 第一个看到的价格影响后续判断 | 先展示高价版本，再展示"超值"版本 |
| **社会规范** | "别人都这样做了"→更愿意做 | 展示"多少人已经买了"/"最受欢迎"标签 |
| **禀赋效应** | 一旦拥有就估值更高 | 先让用户"拥有"（免费试用/定制化） |
| **框架效应** | 同样的东西不同说法感知不同 | "95%成功率"比"5%失败率"好听 |
| **心理账户** | 不同来源的钱花法不同 | 年终奖≠工资，礼物预算≠日常开支 |
| **免费的力量** | 从1元降到0元的吸引力远大于从2元降到1元 | 免费试用的威力比"1元体验"大得多 |

### 5.3 定价心理学要点（《无价》核心）

```
1. 锚定定价：第一个价格是所有后续价格的锚
   → 先展示"原价999"再展示"限时499"——499就成了"划算"

2. 价格是质量信号：当用户没有其他判断依据时，价格越高=质量越好的暗示越强
   → 低价不一定带来更多销量——尤其在不透明的品类

3. 对比效应：A旁边放更贵的B，A就变便宜了
   → "三档定价"（基础/标准/高级）让中间档变成最"值"的选择

4. 数字的心理魔力：
   → 99结尾=便宜（快消品）
   → 00结尾=品质（奢侈品）
   → 精确数字=可信（"497元"比"500元"更像经过计算的真实价格）
```

## L6 · 传播引爆——用户为什么帮你传播

### 6.1 上瘾模型 Hook Model（《上瘾》核心）

```
触发 → 行动 → 多变的酬赏 → 投入 → 触发（循环）

1. 触发：什么东西提醒用户用你？
   外部触发：推送、广告、朋友推荐、图标
   内部触发：无聊→刷抖音、孤独→刷朋友圈、不确定→搜答案

2. 行动：用户用你做什么？
   B=MAT：行为=动机×能力×触发（三者同时存在行为才发生）
   降低行动门槛比增加动机更有效

3. 多变的酬赏：用户得到什么不可预测的回报？
   社交酬赏（点赞/评论）、猎物酬赏（信息/资源）、自我酬赏（成就感/掌控感）

4. 投入：用户在你这里沉淀了什么？
   内容、数据、关注者、声誉、时间→投入越多越离不开（宜家效应）
```

### 6.2 疯传六原则 STEPPS

| 原则 | 含义 | 操作 |
|---|---|---|
| **社交货币** | 分享这个让我看起来更酷/更聪明 | 给用户一个"炫耀"的理由 |
| **诱因** | 环境中的线索提醒用户想到你 | "周五"→提醒分享；"高峰期"→骑共享单车 |
| **情绪** | 高唤醒情绪驱动分享 | 惊奇、兴奋、愤怒（不是悲伤、满足） |
| **公开性** | 能被看到的东西更容易被模仿 | 耳机是白色的→大家知道你在用iPod |
| **实用价值** | 有用的东西人人愿意转 | 省钱技巧、生活妙招、专业干货 |
| **故事** | 信息藏在故事里传播 | 案例＞数据，故事＞论点 |

### 6.3 引爆点三法则

```
个别人物法则：找到你的"联络员、内行、推销员"
  - 联络员：认识所有人的人
  - 内行：对品类极其了解且乐于分享的人
  - 推销员：有说服力、能让人心动的人
  
附着力因素：信息本身让人记住且想行动
  → 你的核心信息能不能在3秒内让人记住？
  
环境威力：环境比人更决定行为
  → 改变环境比改变人更容易（破窗效应：环境乱→犯罪率升）
  → 小群体（150人是邓巴数上限）比大群体更有传播力
```

## 📡 核心工作流

### 工作流A：完善现有商业模式

```
输入：你现在的商业模式描述
     ↓
L1 诊断：画布九要素逐一填写→发现短板和矛盾
     ↓
L2 验证：你的战略能打吗？竞争位置安全吗？
     ↓
L3 优化：利润从哪来？还能从哪来？战略控制指数够吗？
     ↓
L4 创新：有没有被颠覆的风险？蓝海在哪里？
     ↓
L5 人性：用户行为符合设计预期吗？有没有违反行为经济学规律？
     ↓
L6 增长：传播机制有效吗？用户为什么回来？为什么分享？
     ↓
输出：完整的优化方案+画布+下一步行动清单
```

### 工作流B：从点子到商业模式

```
输入：你的点子/想法（一句话就行）
     ↓
Round 1 · 用户是谁？L5先行——用行为经济学理解目标用户
     ↓
Round 2 · 商业模式长什么样？L1画布——九要素逐一讨论填充
     ↓
Round 3 · 怎么验证可行？L4 MVP——最小的实验是什么？
     ↓
Round 4 · 凭什么赢？L2战略——竞争在哪里？蓝海在哪？
     ↓
Round 5 · 怎么赚钱？L3盈利——不止一种赚钱方式
     ↓
Round 6 · 怎么长大？L6增长——传播机制和上瘾设计
     ↓
输出：完整的商业模式画布+战略方案+验证计划+增长路径
```

## ⚠️ 商业模式六大反模式

| # | 反模式 | 为什么死 | 怎么改 |
|---|---|---|---|
| 1 | **爱上方案而不是问题** | 做了一个很酷的东西，但没人需要 | L5从用户开始，别从产品开始 |

## L7 · 概率验证层（v1.1 贝叶斯增强）

> 本层由「概率决策增强模块」注入。来源：《对赌》《停损的胜算》《胜算》《穷查理宝典》《反脆弱》

### 7.1 贝叶斯信念更新循环

你的商业模式基于核心假设。这个假设不是"对"或"错"，它有一个概率。先验概率→拿到新证据→后验概率→持续迭代→越来越接近真相。每一步都问："什么证据会让我更新这个概率？"

### 7.2 止损决策矩阵（《停损的胜算》）

止损判断三步：
1. 设定终止条件（在开始之前）：什么客观条件触发时我放弃？
2. 区分"猴子"（具体问题）和"基座"（根本问题）：修猴子不需要放弃基座
3. 用基础概率做参照：类似项目在全行业的成功率是多少？

### 7.3 事前验尸（Pre-mortem）

假设现在是2年后，这个项目失败了。"我们是因为什么失败的？"→团队每人匿名写3条→汇总排序→针对Top 3提前设防火墙。


### 7.5 狐狸型思维与超级预测（v1.2 补充增强）

> 来源：《超预测》《黑天鹅》《随机生存的智慧》

**狐狸 vs 刺猬**（以赛亚·伯林分类，Tetlock验证）：
- 刺猬：用一个宏大理论解释一切，预测准确度低但自信度高
- 狐狸：从多种角度分析，用概率表达（"60%可能"），持续修正

**概率校准三步**：
1. 用数字替代模糊词：不说"很可能"，说"我判断概率约65%"
2. 费米估算：把大问题拆成小问题，逐段估算后相乘
3. 布里尔分数追踪：记录每一条预测的实际结果，持续校准

**极端斯坦 vs 平均斯坦**（《黑天鹅》）：
- 平均斯坦：个体不会改变整体（身高、体重）→ 正态分布有效
- 极端斯坦：一个极端个案改变一切（财富、点击量）→ 正态分布无效
→ **商业世界在极端斯坦里**——不要用平均思维评估风险

**幸存者偏差**（《随机生存的智慧》）：
你看到的所有"成功案例"都是活下来的样本。失败者不说话。
→ 问自己："我看不到的那些失败案例是什么样的？"

### 7.4 L5增强：芒格人类误判心理学补充

| 心理倾向 | 对商业模式的启示 |
|---|---|
| 激励超级反应 | "给我看激励，我就告诉你结果"——错误激励=模式自毁 |
| 避免不一致倾向 | 用户做了小承诺后，维持一致的动力极强 |
| 被剥夺超级反应 | 失去已有的比从未得到更痛苦（试用后取消=巨大痛苦） |
| 简单联想谬误 | 用户把"包装好看"联想成"产品质量好" |
| 自视过高倾向 | 用户高估自己——"我当然能坚持用这个App" |
| 过度乐观倾向 | 创始人系统性低估风险——做最坏打算 |
